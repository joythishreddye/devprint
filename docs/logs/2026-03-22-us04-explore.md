## Files read

### Project foundation

### Types

| File | Key findings |
|------|-------------|
| `src/types/database.ts` | `Technology` interface has 18 fields. Fields relevant to scoring: `github_stars: number \| null`, `npm_weekly_downloads: number \| null`, `learning_curve: 'beginner' \| 'intermediate' \| 'advanced'`, `community_size: 'small' \| 'medium' \| 'large'`, `maturity: 'emerging' \| 'growing' \| 'mature' \| 'declining'`, `pros: string[]`. All scoring functions must handle `null` gracefully for the numeric fields. |

### PRD (US-04 acceptance criteria)

| Criterion | Notes |
|-----------|-------|
| `compareTechnologies(techA, techB)` returns a structured result | Needs a defined return type — `ComparisonResult` |
| 6 scoring categories | performance, developer_experience, community, learning_curve, ecosystem, maturity |
| Configurable weights summing to 1.0 | Weights belong in a `CATEGORY_WEIGHTS` constant, not hardcoded per-call |
| Scores normalized to 0–10 | A `normalizeScore()` helper needed to clamp and round |
| Overall winner or "tie" | Determined by comparing weighted sums |
| `generateComparisonSummary()` produces human-readable output | Separate function, separate file |
| Pure TypeScript — no UI or DB dependencies | Confirms `src/lib/` placement |
| >90% test coverage | Tests must cover edge cases: null inputs, ties, self-comparison |

## What needs to be built

### New types (`src/types/comparison.ts`)

- `CategoryScore` — per-category result shape
- `ComparisonResult` — full output of `compareTechnologies()`
- `ComparisonSummary` — output of `generateComparisonSummary()`
- `ComparisonCategory` — union type of the 6 category strings
- `CATEGORY_WEIGHTS` — `Record<ComparisonCategory, number>` constant

### New modules (`src/lib/comparison/`)

| File | Responsibility |
|------|---------------|
| `scoring.ts` | Pure functions that convert raw Technology field values to numeric scores |
| `compare-technologies.ts` | Orchestrates scoring, computes weighted overall scores, determines winner |
| `summary.ts` | Converts a `ComparisonResult` into human-readable `ComparisonSummary` |
| `index.ts` | Barrel re-exports for clean import paths |

### Test files (`src/lib/comparison/__tests__/`)

| File | What to cover |
|------|--------------|
| `scoring.test.ts` | All 5 scoring functions + normalizeScore — values, ordering, edge cases (null, 0, very large) |
| `compare-technologies.test.ts` | Result shape, 6 categories present, scores in 0–10 range, winner logic, self-comparison tie |
| `summary.test.ts` | Recommendation text, advantages arrays, tradeoffs, bestFor, tie handling |
