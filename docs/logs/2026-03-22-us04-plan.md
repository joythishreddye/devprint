## Type design (`src/types/comparison.ts`)

```typescript
export interface CategoryScore {
  category: string;
  scoreA: number;
  scoreB: number;
  weight: number;
  normalizedScoreA: number;
  normalizedScoreB: number;
}

export interface ComparisonResult {
  techA: Pick<Technology, 'name' | 'slug' | 'category'>;
  techB: Pick<Technology, 'name' | 'slug' | 'category'>;
  categoryScores: CategoryScore[];
  overallScoreA: number;
  overallScoreB: number;
  winner: 'A' | 'B' | 'tie';
}

export interface ComparisonSummary {
  recommendation: string;
  advantages: { techA: string[]; techB: string[] };
  tradeoffs: string[];
  bestFor: { techA: string; techB: string };
}

export type ComparisonCategory =
  | 'performance' | 'developer_experience' | 'community'
  | 'learning_curve' | 'ecosystem' | 'maturity';

export const CATEGORY_WEIGHTS: Record<ComparisonCategory, number> = {
  performance: 0.2,
  developer_experience: 0.25,
  community: 0.15,
  learning_curve: 0.15,
  ecosystem: 0.15,
  maturity: 0.1,
};
// weights sum: 0.2 + 0.25 + 0.15 + 0.15 + 0.15 + 0.1 = 1.0
```

## Scoring strategy (`src/lib/comparison/scoring.ts`)

Each Technology field maps to a numeric score on a rough 0–10 scale before normalization.

| Function | Input | Logic |
|----------|-------|-------|
| `scoreLearningCurve` | `'beginner' \| 'intermediate' \| 'advanced'` | Lookup: 9 / 6 / 3 — lower curve = higher score |
| `scoreCommunitySize` | `'large' \| 'medium' \| 'small'` | Lookup: 9 / 6 / 3 |
| `scoreMaturity` | `'mature' \| 'growing' \| 'emerging' \| 'declining'` | Lookup: 9 / 7 / 4 / 2 |
| `scoreGitHubStars` | `number \| null` | Tiered thresholds: null → 5, ≥200k → 10, ≥100k → 9, … ≥5k → 5, else 3 |
| `scoreNpmDownloads` | `number \| null` | Tiered thresholds: null → 5, ≥20M → 10, ≥10M → 9, … ≥100k → 5, else 3 |
| `normalizeScore` | `number` | `Math.max(0, Math.min(10, Math.round(score * 10) / 10))` |

## Category score composition (`src/lib/comparison/compare-technologies.ts`)

Each of the 6 categories combines 1–2 scoring functions:

| Category | Formula |
|----------|---------|
| `performance` | `(scoreLearningCurve + (15 - scoreNpmDownloads)) / 2` — lower bundle burden proxy |
| `developer_experience` | `(scoreLearningCurve + min(pros.length * 2, 10)) / 2` |
| `community` | `(scoreGitHubStars + scoreCommunitySize) / 2` |
| `learning_curve` | `scoreLearningCurve` directly |
| `ecosystem` | `(scoreNpmDownloads + scoreGitHubStars) / 2` |
| `maturity` | `scoreMaturity` directly |

Overall score = `sum(normalizedScore * weight)` across all 6 categories, rounded to 1 decimal.

Winner: `scoreA > scoreB → 'A'`, `scoreB > scoreA → 'B'`, equal → `'tie'`.

## Summary generation (`src/lib/comparison/summary.ts`)

`generateComparisonSummary(result)` logic:

1. Loop `categoryScores` — if `normalizedScoreA > normalizedScoreB`, add to `advantagesA`; vice versa for `advantagesB`
2. Tradeoffs = categories where `abs(scoreA - scoreB) >= 1.5` — these are the meaningful gaps
3. If no tradeoffs meet the threshold, fall back to "very close across all categories"
4. `recommendation`:
   - tie → "either is a strong choice"
   - winner → "{winner} edges ahead, but {loser} has clear strengths in specific areas"
5. `bestFor` = derive from the top 2 advantages, or fall back to category description
