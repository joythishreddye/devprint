## Problems found during exploration

### 1. Test runner broken for component tests

Running `npm run test` against a component test file fails with:

```
Error [ERR_REQUIRE_ESM]: require() of ES Module
  node_modules/@csstools/css-calc/dist/index.mjs
```

Root cause: `vitest.config.ts` is loaded as CommonJS (no `"type": "module"` in `package.json`). Vitest 3.x uses Vite internally which requires ESM. Additionally, the `jsdom` v27 environment pulls in `@asamuzakjp/css-color`, which tries to `require()` an ESM-only package.

Fix required before writing component tests:
1. Rename `vitest.config.ts` → `vitest.config.mts` to force ESM loading
2. Replace `jsdom` with `happy-dom` (no CJS/ESM conflict)
3. Set default environment to `node`; apply `happy-dom` only to `src/components/**`

### 2. No existing components

`src/components/` does not exist. All UI components for US-05 must be created from scratch.

### 3. No fixture data

No hardcoded `Technology` objects exist outside the test files. The compare page cannot fetch from Supabase (not connected yet), so a `fixtures.ts` file is needed to provide demo data.
