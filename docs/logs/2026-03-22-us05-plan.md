## Acceptance criteria

- [ ] User can select two technologies from dropdowns at `/compare`
- [ ] Page displays side-by-side cards with stats and pros/cons
- [ ] Category scores shown as a visual bar chart
- [ ] Recommendation summary displayed below the comparison
- [ ] Comparison URL is shareable (`?a=react&b=svelte`)

## Component breakdown

### `CategoryScoreBar` ‚Äî server component

Accepts `CategoryScore[]` from the comparison engine and renders one row per category with proportional bar widths.

**Props:**
```typescript
interface CategoryScoreBarProps {
  scores: CategoryScore[];
  nameA: string;
  nameB: string;
}
```

**Note on bar widths:** Tailwind v4 JIT cannot generate dynamic width classes from runtime values. Bar widths must use `style={{ width: '${score * 10}%' }}`. All surrounding layout still uses Tailwind.

---

### `ComparisonSummaryPanel` ‚Äî server component

Renders the human-readable output from `generateComparisonSummary()`.

**Props:**
```typescript
interface ComparisonSummaryPanelProps {
  summary: ComparisonSummary;
  nameA: string;
  nameB: string;
}
```

Sections: recommendation ‚Üí advantages (two columns) ‚Üí tradeoffs ‚Üí best-for (two columns).

---

### `TechnologySelector` ‚Äî `"use client"` component

Fully controlled dual `<select>` with no internal state. Parent owns the selected values and passes callbacks.

**Props:**
```typescript
interface TechnologySelectorProps {
  technologies: Pick<Technology, 'name' | 'slug'>[];
  selectedSlugA: string;
  selectedSlugB: string;
  onChangeA: (slug: string) => void;
  onChangeB: (slug: string) => void;
  validationError: string | null;
}
```

Keeping `TechnologySelector` fully controlled (no `useRouter` dependency) means its tests require no Next.js mocks.

---

### `ComparisonClient` ‚Äî `"use client"` thin boundary

Owns the URL update logic. Uses `useRouter().replace()` with `{ scroll: false }` so the page doesn't jump on selection change. Passes callbacks into `TechnologySelector`.

**Props:**
```typescript
interface ComparisonClientProps {
  technologies: Pick<Technology, 'name' | 'slug'>[];
  initialSlugA: string;
  initialSlugB: string;
  validationError: string | null;
}
```

---

### `src/lib/comparison/fixtures.ts`

Four complete `Technology` objects (React, Vue.js, Svelte, Next.js) with realistic stats. Exported as:
- `FIXTURE_TECHNOLOGIES: Technology[]`
- `FIXTURE_TECHNOLOGIES_BY_SLUG: Record<string, Technology>` ‚Äî for O(1) slug lookup in the page

---

### `src/app/compare/page.tsx` ‚Äî server component

```
searchParams: Promise<{ a?: string; b?: string }>   ‚Üê async in Next.js 16

slugA = searchParams.a ?? ''
slugB = searchParams.b ?? ''
techA = FIXTURE_TECHNOLOGIES_BY_SLUG[slugA] ?? null
techB = FIXTURE_TECHNOLOGIES_BY_SLUG[slugB] ?? null

validationError = slugA && slugB && slugA === slugB
  ? 'Please select two different technologies'
  : null

result  = techA && techB && !validationError ? compareTechnologies(techA, techB) : null
summary = result ? generateComparisonSummary(result) : null
```

Renders: `ComparisonClient` ‚Üí side-by-side `TechCard` ‚Üí `CategoryScoreBar` ‚Üí `ComparisonSummaryPanel`
